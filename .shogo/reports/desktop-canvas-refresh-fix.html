Canvas refresh pill missing on Desktop

Root-cause analysis, fix, before/after, tests, E2E.

The symptom

Before — Desktop

After every agent rebuild, the canvas silently kept showing stale content. No pill, no toast, no signal.
( no toast rendered )

After / Cloud — the working state

SSE rebuild event → workspace iframe shows pill → user clicks Refresh.
Update available ✕

Root cause

The pill is rendered by packages/agent-runtime/static/canvas-bridge.js. The agent-runtime:

  1. Reads the file at startup via readFileSync(join(__dirname,'..','static','canvas-bridge.js'))
  2. Serves the bytes at GET /agent/canvas/bridge.js
  3. Injects <script src="/agent/canvas/bridge.js" defer> into every workspace HTML response
  4. Inside the iframe, the bridge opens EventSource('/agent/canvas/stream') and shows the pill on rebuild events

On Cloud, static/ sits next to the runtime, so step 1 succeeds. On Desktop, apps/desktop/scripts/bundle-api.mjs copied everything except the static/ folder into resources/. The runtime then silently fell into its console.warn + empty-IIFE-stub fallback. No throw, no failed test, no failed build — the only signal was a buried stderr warning.

Why a stub, why silent?

// packages/agent-runtime/src/canvas-bridge.ts (was inline in server.ts)
export function loadCanvasBridgeSource(path: string = CANVAS_BRIDGE_PATH): string {
  try {
    return readFileSync(path, 'utf-8')
  } catch (err) {
    console.warn(`[canvas-bridge] Failed to load ${path}:`, (err as Error).message)
    // Empty IIFE keeps the route honest (200 with valid JS) even when the
    // bridge file is missing — the canvas just won't show update toasts.
    return CANVAS_BRIDGE_MISSING_STUB
  }
}

Soft-fail is the right behavior — a missing asset shouldn't 500 the iframe's script tag and break the whole canvas. The bug is one layer up: the bundle script wasn't shipping the asset. That's where the fix lives.

The fix — patch vs root

rejected   patch fix

  • Inline the bridge JS as a string literal in server.ts
  • Add a second fallback path (e.g. resources/canvas-runtime/canvas-bridge.js)
  • Conditional Desktop-only branch in the loader

All of these treat the symptom (missing bytes at runtime) instead of the cause (missing bytes in the bundle output).

applied   root fix

  • bundle-api.mjs copies static/ → resources/static/
  • forge.config.ts ships ./resources/static as extraResource
  • static added to bundle cleanup list so rebuilds stay deterministic
  • Loader unchanged in behavior; extracted to canvas-bridge.ts for testability

The runtime keeps its lookup formula. The bundle now satisfies it. Dev mode unaffected.
