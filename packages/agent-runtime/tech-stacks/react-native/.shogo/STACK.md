# Stack: React Native (bare)

This project is a bare **React Native** app with TypeScript. Unlike the
`expo-app` stack, there is no Expo SDK / Expo Router — this is the lower-level
RN setup that compiles into native iOS / Android projects directly.

The workspace also ships with a **Hono + Prisma** API server at the root
(`server.tsx`, `prisma/schema.prisma`, `shogo.config.json`). The mobile
client and the API live side-by-side in the same package; the runtime
spawns the bundler and the API server together.

## Runtime

- **Bundler:** Metro, supervised by the runtime's PreviewManager. **Never run
  `react-native start` or `npx react-native start` from `exec`** — Metro is
  long-lived and the bundler is started for you.
- **Web preview:** bare RN does **not** produce a web bundle the way Expo
  does. Cloud preview surfaces a "device preview only" indicator; iterate
  locally with the Metro tunnel + a connected simulator/device.
- **API server:** `server.tsx` is generated by `bun x shogo generate` from
  `prisma/schema.prisma` + `shogo.config.json`, and runs on port 3001
  (configurable in `shogo.config.json`). Custom routes live in
  `custom-routes.ts`.

## Files

- `index.js` — RN registry entry. Don't rename.
- `App.tsx` — root component.
- `app.json` — RN config (name / displayName).
- `babel.config.js`, `metro.config.js` — bundler config; usually fine as-is.
- `prisma/schema.prisma` — single source of truth for the API's data model.
  Edit this and the runtime regenerates `server.tsx` + per-model CRUD routes.
- `custom-routes.ts` — non-CRUD endpoints (aggregations, proxies, webhooks).
  The agent-facing surface; mounts under `/api/`.
- `shogo.config.json` — generation config (port, CORS, paths).

## Conventions

- Use functional components and hooks.
- Use `react-native` primitives (`View`, `Text`, `Pressable`) — not raw
  `div` / `span`.
- For navigation, use `@react-navigation/native` if added.
- Persistent state on-device: `@react-native-async-storage/async-storage`.
- For server-side data, edit `prisma/schema.prisma` and let the SDK
  regenerate the CRUD layer; call it from RN with the generated client.

## Forbidden commands

- `react-native start`, `npx react-native start`, `metro` — the bundler is
  owned by PreviewManager.
- Editing `server.tsx` directly — it is regenerated from `shogo.config.json`.
  Put custom logic in `custom-routes.ts`.
- `kill`, `pkill` on the bundler or API process.
